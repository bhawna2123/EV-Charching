# EV_ChargingStation_MS

FRONT-END FEATURES
• React native: A mobile app development framework for creating high performance,
graphically appealing apps for both Android and iOS platforms
• Google Maps API: A mapping platform that provides developers with location data
and mapping services to integrate into their applications
• Google Place API: The Places API (application programming interface) is a service
that provides information about places using HTTP requests
• Google Direction API: It is a set of APIs (application programming interfaces) that
provide the communication bridge to Google’s various services.
• Reservation system: A system that allows consumers to reserve charging spots in
advance for their electric vehicles
• Clerk dev: It is used for viewing the login details of user.
• Interoperable reservation system: A reservation system reference architecture that
ensures interoperability across different EV charging stations.
2.2 BACK-END FEATURES

JavaScript
As a widely used programming language, JavaScript will serve as the backbone for
server-side logic and overall application functionality.
Node.js
A runtime environment for executing JavaScript on the server-side, Node.js will
power the backend server, allowing for efficient and event-driven handling of server
requests and responses.
APIs
Integrate APIs to fetch real-time data about charging station locations, availability,
and pricing. This can be done using Google maps API, Google places API(new).

Firebase
Firebase offers a comprehensive set of tools and services to streamline various aspects of
the application. Firstly, Firebase Real-time Database facilitates the management of
charging station information, slot availability, and booking statuses in real time, ensuring
users have up-to-date information. Secondly, Firebase Authentication provides secure
user authentication, enabling features like sign-up, login, and account management. This
ensures that user data and transactions are protected. Thirdly, Firebase Cloud Functions
can be utilized to handle backend logic for processing booking requests, updating
database entries, and sending notifications to users.
